# Weakness model

Explicit `unknown`, `learning` with unresolved questions, repeated weak events, and configured prerequisite blockers are weakness signals. `unrated`, exposure-only, a single explanation, or an unsupported claim of knowledge are insufficient evidence and trigger diagnosis rather than a weakness label. A due `known`/`mastered` item triggers short retrieval review.

Priority is an explainable heuristic: explicit weakness + repeated confusion + due review + Mission relevance + prerequisite value − recent overpractice. It is not a cognitive measurement. Ranking is descending priority then stable concept ID. Missing prerequisite relations are reported, not invented.
